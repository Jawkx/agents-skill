# Quality Checks

Run these checks before delivering the HTML file.

## Factual Checks

- Every number, name, and behavior description is backed by a file, document, or command output.
- Assumptions are labeled as assumptions.
- The opening summary matches the evidence shown later in the page.

## Visual Checks

- The first screen communicates the main point quickly.
- Sections have clear hierarchy; not every card looks equally important.
- No text overflows its container.
- Tables remain readable on narrow screens or inside a scroll wrapper.
- Code blocks wrap or scroll intentionally instead of breaking layout.

## Diagram Checks

- Mermaid output, if used, is readable at the default size.
- Labels are concise and do not collide badly.
- Hybrid pages keep the overview diagram simple enough to scan.

## Responsive Checks

- The page works on both desktop and mobile widths.
- Side-by-side panels stack cleanly when space is tight.
- Sticky or anchored navigation does not cover content.

## Finish

- Save to the chosen local path.
- Open the file when supported.
- Tell the user what the page contains, where it lives, and any important limitations.
